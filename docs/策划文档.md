### 1.1.0（相对 1.0.0）

- **规则与体验**
  - 统一客户端与服务器房间状态关键字为 `waiting`（原客户端使用 `menu` 导致等待界面逻辑错误）。
  - 砖块得分改为归属「当前球最后一次击中挡板」的玩家，取消随机加分。
  - 多人加入或离开时，所有挡板按人数**重新均匀排布**，避免重叠或布局错乱。
- **缺陷修复**
  - 服务器补充 `BRICK_OFFSET_LEFT` 与 `PADDLE_SPEED`，修复砖块 X 坐标为 `NaN`、键盘移动挡板无效等问题。
  - 玩家对象绑定 WebSocket 引用，断线时正确移出房间并刷新挡板位置。
  - 客户端本地「最高分」与当前总分比较改为**数值比较**，避免字符串字典序错误。
  - 主界面按钮：`gameover` / `win` 时「再玩一次」发送 `restart`；`paused` 时主按钮发送 `pause` 以继续，与服务器指令一致。
- **客户端健壮性**
  - WebSocket 消息 `JSON.parse` 增加防护；断线后**指数退避自动重连**（带代数校验，避免旧连接回调干扰）；支持手动点击「重连」。
  - Canvas `roundRect` 不可用时回退为 `rect`，避免旧浏览器报错。
- **工程**
  - `package.json` 声明 `ws` 依赖，便于 `npm install` 一键安装。

## 1. 产品概述

浏览器多人协作打砖块：多名玩家各自控制一条挡板，共用一颗球与一面砖墙，生命与分数由服务器同步。适合局域网或部署在同一主机上的好友通过同一 URL 加入。

---

## 2. 核心规则

| 项目 | 说明 |
|------|------|
| 人数 | 1～4 人同房间 |
| 生命 | 初始 3；球落底扣 1；为 0 时 `gameover` |
| 胜利 | 清空所有砖块 → `win` |
| 得分 | 每击碎一块砖 **+10**，加给**当前球最后一次击中挡板**的玩家；发球时归属发球点所在玩家 |

**状态机（服务器权威）：**  
`waiting` →（开始）→ `playing` ↔ `paused` → `gameover` / `win`；`restart` 回到 `waiting` 并重置生命、砖块与分数。

---

## 3. 操作说明

- **移动挡板：** ← → 或 A D；鼠标在画布上移动同步控制（与键盘二选一或组合，以服务器每帧合并为准）。
- **暂停：** 空格或界面「暂停」；暂停时主菜单按钮文案为「继续游戏」，发送 `pause` 切换。
- **开始：** 至少一名玩家在房间内，点击「开始游戏」发送 `start`。
- **再玩一局：** 游戏结束或通关后点击「再玩一次」发送 `restart`。
- **断线：** 自动重连（退避）；亦可点击「重连」立即尝试新的连接与加入流程。

---

## 4. 技术架构（简述）

- **HTTP：** 提供 `index.html`、`style.css`、`game.js`。
- **WebSocket：** 客户端发送 `join` / `input` / `command`；服务器广播 `gameState`（含球、砖块、玩家挡板与分数、生命、状态）。
- **模拟：** 球运动、挡板移动、碰撞与得分均在服务器 `gameLoop`（约 60Hz）中计算，客户端仅渲染与上报输入。

---

## 5. 非目标与限制（当前版本）

- 多房间匹配、账号系统、反作弊加固未实现。
- 本地「最高分」为浏览器 `localStorage`，仅本机、本域名有效。
- 单进程单房间：所有连接共享同一 `game` 状态。
